# Wallet: Verification + Real Money যোগ করার গাইড

## ১. টাকা রিসিভ/উইথড্রয়ের জন্য ভেরিফিকেশন (কিভাবে করবেন)

এপে **Withdraw** সেকশনে এখন এই ফ্লো আছে:

- ব্যবহারকারী প্রথমে **Verify now** ক্লিক করে।
- মডালে নাম, ফোন ও **পেমেন্ট মেথড** দেয়:
  - **Bank account** – ব্যাংক নাম, অ্যাকাউন্ট নম্বর, IFSC/ব্রাঞ্চ কোড
  - **UPI** – UPI ID (GPay, PhonePe, Paytm)
  - **bKash / Nagad / Rocket** (BD) – ওয়ালেট মোবাইল নম্বর
- **Submit & verify** করলে ডেটা লোকালস্টোরেজে সেভ হয় এবং “Account verified ✓” দেখায়।
- এর পরই **Withdraw** বাটন দিয়ে অ্যামাউন্ট উইথড্রয় করা যায় (ডেমোতে অ্যালার্ট; প্রডাকশনে ব্যাকেন্ড দিয়ে টাকা পাঠাতে হবে)।

**প্রডাকশনে ভেরিফিকেশন কিভাবে করবেন:**

1. **Backend API** বানান:  
   `POST /api/wallet/verify-payout` – নাম, ফোন, payout type, bank/UPI/wallet details সেভ করুন (ডাটাবেজে, এনক্রিপ্টেড)।
2. **KYC (optional কিন্তু ভালো):**  
   প্রথম উইথড্রয়ের আগে ID (প্যান/আধার/পাসপোর্ট) ও সেলফি আপলোড নিতে পারেন; তৃতীয় পক্ষ সেবা (যেমন Razorpay KYC, Signzy) ব্যবহার করা যায়।
3. **OTP / ইমেইল ভেরিফিকেশন:**  
   ফোন বা ইমেইলে OTP দিয়ে নিশ্চিত করুন যে নম্বর/ইমেইল ইউজারের।
4. **প্রথম payout যাচাই:**  
   ছোট অ্যামাউন্ট (যেমন ৳1) পাঠিয়ে অ্যাকাউন্ট সঠিক কিনা চেক করা যায়।

ফ্রন্টএন্ডে যা আছে (verify modal + localStorage) সেটা শুধু ডেমো; প্রডাকশনে ওই ফর্মের ডেটা backend এ পাঠিয়ে সেভ ও ভেরিফাই করুন।

## ২. রিয়েল টাকা কিভাবে অ্যাড করবেন

এখন **Add money** পেজে UPI, Paytm, PhonePe, Bank, Card ইত্যাদি সিলেক্ট করে **Pay** দিলে শুধু **সিমুলেশন** হয় (কোনো রিয়েল টাকা কাটে না; ব্যালান্স লোকালস্টোরেজে বাড়ে)।  
রিয়েল টাকা অ্যাড করতে নিচের স্টেপগুলো লাগবে।

### যা লাগবে

- **Backend (সার্ভার)**  
  Node.js, PHP, Python ইত্যাদি যেকোনো একটা। ফ্রন্টএন্ড শুধু amount + payment method পাঠাবে; ব্যাকেন্ড পেমেন্ট ক্রিয়েট ও ভেরিফাই করবে।

- **পেমেন্ট গেটওয়ে অ্যাকাউন্ট**  
  রিজিয়ন অনুযায়ী একটা বা একাধিক:

| Region        | Gateway (উদাহরণ)        | ব্যবহার           |
|---------------|--------------------------|--------------------|
| India (IN)    | Razorpay, Paytm Business, PhonePe for Business | UPI, Cards, Net Banking, Wallets |
| Bangladesh (BD) | bKash, Nagad, Rocket (SSLCommerz ইত্যাদি) | মোবাইল ওয়ালেট, কার্ড |
| International | Stripe, PayPal           | কার্ড, পেপ্যাল     |

### সাধারণ ফ্লো (রিয়েল টাকা অ্যাড)

1. ইউজার অ্যামাউন্ট ও পেমেন্ট মেথড সিলেক্ট করে **Pay** ক্লিক করে।
2. **Frontend** ব্যাকেন্ডে রিকোয়েস্ট পাঠায়, যেমন:  
   `POST /api/wallet/create-order`  
   Body: `{ amount: 500, currency: "INR", method: "upi", upiId: "user@paytm" }` (অথবা আপনার API ফরম্যাট)।
3. **Backend** গেটওয়েতে অর্ডার/পেমেন্ট ক্রিয়েট করে (যেমন Razorpay Order বা Payment Link) এবং ফ্রন্টএন্ডকে **payment_id / order_id / redirect URL** রিটার্ন করে।
4. **Frontend** ইউজারকে গেটওয়ের পেজে রিডাইরেক্ট করে অথবা (UPI/কاردের জন্য) সেই পেজেই গেটওয়ের SDK দিয়ে পেমেন্ট সম্পন্ন করে।
5. গেটওয়ে **webhook** বা **success redirect** দিয়ে ব্যাকেন্ডকে জানায় পেমেন্ট সাকসেস হয়েছে।
6. **Backend** ওয়ালেট ব্যালান্স আপডেট করে (ডাটাবেজে) এবং ফ্রন্টএন্ডকে সাকসেস রেসপন্স দেয়।
7. **Frontend** সাকসেস পেজে বা ওয়ালেট পেজে রিডাইরেক্ট করে এবং আপডেটেড ব্যালান্স দেখায়।

### কোডে কী পরিবর্তন করবেন (`wallet-add.html`)

- **Pay** বাটনের ক্লিক হ্যান্ডলারে:
  - এখন যেখানে সরাসরি `setTakaAdded(...)` ও `setCoins(...)` এবং রিডাইরেক্ট করা হয়, সেখানে **আগে** আপনার backend API কল করুন।
  - উদাহরণ:  
    `fetch('/api/wallet/create-order', { method: 'POST', body: JSON.stringify({ amount, method, upiId, ... }) })`  
    → রেসপন্সে `payment_url` বা `order_id` পেলে ইউজারকে সেই URL এ নিয়ে যান অথবা গেটওয়ের SDK চালু করুন।
  - পেমেন্ট সাকসেস হওয়ার পর ব্যাকেন্ড থেকে ব্যালান্স নিয়ে ওয়ালেট পেজে রিডাইরেক্ট করুন; অথবা ব্যাকেন্ড ওয়ালেট আপডেট করে দিলে ফ্রন্টএন্ড শুধু ওয়ালেট পেজ লোড করলেই নতুন ব্যালান্স দেখাবে।

- **রিয়েল টাকা চালু থাকলে** লোকালস্টোরেজের বালান্স ব্যবহার না করে সবসময় **backend থেকে ব্যালান্স** লোড করুন (যেমন `GET /api/wallet/balance`)।

## সংক্ষেপে

| কাজ                    | এখন (ডেমো)                    | প্রডাকশনে কী করবেন |
|------------------------|--------------------------------|----------------------|
| টাকা রিসিভের ভেরিফিকেশন | মডালে নাম/ফোন/ব্যাংক/UPI সেভ (localStorage) | Backend এ সেভ + KYC/OTP + প্রথম payout টেস্ট |
| উইথড্রয়               | অ্যালার্ট + লোকাল ব্যালান্স কমানো   | Backend দিয়ে ব্যাংক/UPI/wallet এ টাকা পাঠানো |
| টাকা অ্যাড (রিয়েল)    | সিমুলেশন, ব্যালান্স localStorage | Backend + Razorpay/bKash/Stripe ইত্যাদি গেটওয়ে |

রিয়েল টাকা ও রিয়েল ভেরিফিকেশনের জন্য **backend + payment gateway + (optional) KYC provider** যোগ করলেই হবে; ফ্রন্টএন্ড ফ্লো ইতিমধ্যে মিলে গেছে।
